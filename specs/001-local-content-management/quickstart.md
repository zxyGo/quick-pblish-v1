# Quickstart & Validation: 本地内容基座

**Feature**: 001-local-content-management | **Date**: 2026-06-21

本文件是端到端验证指南——证明功能按 spec 工作。实现细节归 tasks.md 与实现阶段。

## 前置条件

- Node.js LTS + 包管理器（pnpm/npm）
- Rust 工具链（rustup，stable）
- 平台 Tauri 依赖（参见 Tauri 官方 prerequisites：Windows 的 WebView2 / macOS Xcode CLT / Linux webkit2gtk 等）

## 启动开发环境

```bash
pnpm install
pnpm tauri dev      # 启动 Vue 前端 + Rust 后端的桌面开发窗口
```

## 验证场景（对应用户故事）

### 场景 1：创作并保存一篇文章（US1 / P1）

1. 启动应用，按引导选择一个空文件夹作为工作目录。
2. 新建文章，输入标题与正文（含标题、列表、代码块），观察右侧预览实时渲染（FR-010/FR-011）。
3. 保存。
4. **预期**：工作目录出现一个 `.md` 文件，头部含 YAML front matter（title/tags/created/updated），正文与输入一致。
5. 关闭并重开应用 → 文章仍在，内容一致（SC-002）。
6. 修改后不保存直接关闭 → 出现"未保存更改"提示（FR-007）。

### 场景 2：选择与切换工作目录（US2 / P1）

1. 在设置/侧栏切换到另一个文件夹。
2. **预期**：文章列表与文件树更新为新目录内容，原目录文件不变（FR-002）。
3. 在应用外把当前工作目录改名/移走，重启应用。
4. **预期**：应用提示目录不可访问并引导重新选择，不崩溃（FR-003）。

### 场景 3：浏览与组织文件树（US3 / P2）

1. 在文件树新建文件夹，将一篇文章移入，再重命名。
2. **预期**：磁盘真实结构随之改变，文章仍可正常打开（FR-013）。
3. 删除一个文件 → 确认后文件进入系统回收站，可在系统回收站找到（FR-006/FR-013）。
4. 放一张图片进目录 → 文件树展示该素材（FR-014）。

### 场景 4：列表、元数据与检索（US4 / P2）

1. 准备多篇文章（含不同标签）。
2. 打开列表 → 每篇显示标题、创建/修改时间、标签（FR-015）。
3. 输入关键字检索 → 仅显示标题/标签/正文匹配项（FR-017）。
4. 给某篇加标签并保存 → 标签写入 front matter，可用于过滤（FR-016）。
5. 按"修改时间"排序 → 列表重排（FR-017）。
6. 手动损坏一篇文章的 front matter → 列表仍能加载该篇（降级展示），不报错中断（FR-018）。

### 场景 5：插图与冲突检测

1. 在编辑器插入本地图片 → 图片被复制进 `assets/`，正文为相对路径引用（FR-014a）。
2. 打开一篇文章，在应用外用别的编辑器修改并保存该文件，回到应用点保存。
3. **预期**：弹窗提示外部修改，可选 覆盖 / 放弃本地并重载 / 另存为（FR-019）。

### 场景 6：派生缓存可重建（FR-008a）

1. 关闭应用，删除应用数据目录中的 SQLite 缓存文件。
2. 重启应用。
3. **预期**：列表自动从 Markdown 文件重建，内容完整，无数据丢失。

## 离线验证（SC-005）

- 断网后重复场景 1、3、4 的核心操作（新建/编辑/保存/组织/检索）→ 全部可用。

