# Quickstart / 验证指南: 多平台发布

证明本功能端到端可用的可运行验证场景。前置依赖 001（已有工作目录与文章）。
契约细节见 [contracts/](./contracts/)，数据模型见 [data-model.md](./data-model.md)，不在此重复。

## 前置条件

- 已按 001 选择一个工作目录，且其中至少有一篇含标题、代码块、引用与**本地 `assets/` 图片**的文章。
- **SC-003 参照文章（固定样本，保证一致率门禁可复现）**：准备一篇 `reference.md`，必须包含以下 7 类样式元素各至少一处——一级/二级标题、有序与无序列表、行内代码、代码块（带语言）、引用块、加粗/斜体、一张本地图片。SC-003 的"抽样核对"即对这 7 类元素逐一比对草稿与预览，一致计数 / 7 ≥ 95%（即 7 类全一致）。
- 拥有微信公众号（及可选知乎/掘金）的可登录账号。
- 已安装本切片新增后端依赖（`keyring`、`aes-gcm`、`rand`，见 plan.md）。

## 启动

```bash
pnpm install
pnpm tauri dev
```

## 场景 1：连接平台（US1 / FR-001~006）

1. 打开"平台"面板（`PlatformPanel`），三平台均显示 `Disconnected`。
2. 点击"微信公众号 → 连接"，在弹出的内嵌 WebView 中扫码登录。
3. **预期**：面板该平台变为 `Connected` 并显示账号名称。
4. 完全退出并重启应用。
5. **预期**：公众号仍为 `Connected`，无需再次登录（会话已加密落盘并回灌）。
6. 点击"断开连接"。
7. **预期**：状态回到 `Disconnected`；应用数据目录无明文凭据、OS keyring 对应密钥已删除。

## 场景 2：单平台一键存草稿（US2 / FR-007~012, SC-001/003/005）

1. 在文章列表选中目标文章，打开 `PublishDialog`，勾选已连接的"微信公众号"，点击"同步"。
2. **预期（≤60s）**：提示"同步成功"，并提供"前往平台查看草稿"链接。
3. 打开公众号草稿箱。
4. **预期**：新增一条草稿；对参照文章的 7 类样式元素（见前置条件）逐一比对，一致率 ≥95%（SC-003）；
   图片正常显示且 `src` 指向公众号自家地址，**正文无任何本地文件路径**。
5. **（SC-001 计时）** 记录"点击同步 → 草稿出现"耗时，应 ≤60s。
6. 在未登录知乎时勾选"知乎"并同步。
7. **预期（FR-012 双层）**：前端预拦截或后端将该平台 job 判 `Failed`(reason=`Auth`) 并提示"需先登录知乎"，非静默失败。

## 场景 3：多平台批量 + 失败隔离 + 重试（US3 / FR-013~016a）

1. 连接公众号与掘金；选中文章，勾选两者，点击"同步"。
2. **预期**：`SyncResultList` 逐平台显示进度，最终各得到一条草稿。
3. 制造一个平台失败（如同步前断网该平台，或令其登录态过期）。
4. **预期**：失败平台标记 `Failed` 且给出可读原因；另一平台仍 `Success`，互不影响。
5. 点击失败平台的"重试"。
6. **预期**：仅该平台重新执行并成功，在平台**新建**一条独立草稿（不覆盖此前草稿）。

## 场景 4：图片失败"全有或全无"（FR-010a / SC-005）

1. 选一篇含多张本地图片的文章，人为让其中一张不可上传（如临时改坏该图）。
2. 触发同步到公众号。
3. **预期**：该平台判 `Failed` 并提示图片上传失败原因；公众号草稿箱**不出现**不完整/坏图草稿。
4. 修复图片后重试。
5. **预期**：同步成功，所有图片均替换为平台 URL。

## 场景 5：同步历史（US4 / FR-018）

1. 对某文章完成若干次同步（含成功与失败）。
2. 打开该文章的 `SyncHistory`。
3. **预期**：按时间倒序列出每次同步的平台、时间、结果（成功含草稿链接，失败含原因）。

## 自动化测试入口

- 后端：`cargo test`
  - 会话加解密往返（明文不落盘、密钥取自 keyring 抽象）
  - 各 adapter `transform_html` 平台化结果（公众号保留内联样式）
  - `sync.rs` 编排：用 `MockAdapter` 验证逐平台隔离、失败不阻断、重试新建、图片失败整篇失败
  - `history` 读写与按文章倒序查询
- 前端：`pnpm test`（Vitest）
  - `publish` store 状态机（Pending/Running/Success/Failed、进度事件归并）
  - `render.ts` 用 `@md/core` 渲染产出非空内联样式 HTML
  - `SyncResultList` 依据 job 状态渲染进度/重试入口
